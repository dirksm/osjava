What is this all about?

This is a MetaJavadoc system. It consists of a wrapper around a set of javadocs.

Why does it exist?

First and foremost, to put all of Jakarta Commons into a single javadoc, possibly all of Apache/Jakarta if desired. Also should be useful for other locations; osjava, codehaus etc.

How does it work?

The current version scrapes from a list of javadoc urls and collates a small set of pages around those pages.

Future?
